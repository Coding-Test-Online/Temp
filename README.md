## Github

## PR 방식
### 1. repository를 fork
### 2. `{본인 이름}/{주차}/{문제 이름}` 경로에 풀이 코드 작성
```java
ex) yukeon/1주차/055-jump-game/055-jump-game.java
```
### 3. 모든 파일과 디렉토리의 이름은 소문자와 하이픈(’-’)으로 작성
### 4. 문제를 다 풀면 PR 작성
### 5. PR 양식 : `{주차}_{이름}`
```java
ex) yukeon/1주차/055-jump-game/055-jump-game.java
```
### 6. PR을 더이상 수정하지 않는다면 `automerge` 라벨 적용
### 7. 자동 merge
